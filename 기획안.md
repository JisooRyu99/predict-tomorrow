# 내일을 맞혀라 — 기획안

> **앱 이름: 내일을 맞혀라** (전국민 예측 게임). 앱인토스(Apps in Toss) 미니앱, 밸런스 게임 다음 작품.
> 목표: **체류시간 + 재방문 + 광고 수익** 극대화.
> ⚠️ appName은 콘솔 등록 후 수정 불가 → 콘솔 값과 정확히 일치시킬 것.

---

## 1. 한 줄 콘셉트

매일 현실(환율·주가·스포츠 등)을 예측하고, 다음 날 정답을 확인하며 **점수·순위·연속 적중 스트릭**을 쌓는 게임.

---

## 2. 핵심 루프 (체류·재방문의 엔진)

```
예측한다 → (기다림) → 다음 날 정답 공개 → 점수/순위 변동 확인 → 또 예측한다
```

- **재방문**: "내 예측이 맞았나?" 확인이 다음 날 들어올 절대적 이유 → 데일리 방문 강제
- **체류**: 여러 종목 예측 + 랭킹·통계·지난 기록 구경
- **콘텐츠가 현실이라 복제 불가** — 남이 구조를 베껴도 똑같은 게임이 안 됨

---

## 3. 보상 설계 (중요 — 사행성 회피)

| 층 | 무엇 | 비용/수익 | 사행성 |
|---|---|---|---|
| **게임 내 보상 (메인)** | 적중 시 점수·랭킹·연속 적중 스트릭·뱃지 (가상) | 무료 | 안전 ✅ |
| **광고 (수익원)** | "예측권 추가 / 결과 미리보기 / 랭킹 부스트" → 리워드 광고 | 수익 발생 💰 | 안전 ✅ |
| **토스 포인트 (양념)** | "출석 7일", "첫 예측" 등 참여 보상으로만 소량 | 내 비용 | 적중과 무관해야 안전 |

> 🔑 **원칙: "맞히면 돈"이 아니라 "맞히면 점수/순위". 돈은 광고로 번다.**
> 미래 예측 적중 → 현금/환금성 보상은 사행성(도박) 규제 리스크. 토스 포인트는 적중 보상이 아니라 출석·온보딩 유도에만 쓴다.

### 🚨 네이밍 규칙 (토스 정책 — 위반 시 반려)
- **자체 재화를 "포인트"라고 부르면 안 됨** ('토스 포인트' 오인). → 우리는 **"점수"**로 통일. UI 어디에도 "포인트" 금지.
- **"출금 / 인출"** 등 현금화 연상 용어 금지. 토스 포인트로 전환되는 경우에만 "토스 포인트 지급" 표기 허용.

### 토스 포인트 지급 (선택 — 출시엔 빼는 걸 추천)
- 지급은 **비즈월렛 예산에서 차감 = 내 비용.** 예산 소진 시 `4109` 에러 + 지급 실패 → CS 이슈.
- **"무조건 무료" 원칙이면 출시 버전엔 토스 포인트 미사용.** 점수/랭킹 + 광고만으로 운영. 토스 포인트는 나중에 출석 이벤트 같은 성장 레버로만 옵션.
- 쓸 경우 SDK: 게임 카테고리 `grantPromotionRewardForGame` / 비게임 `grantPromotionReward` (카테고리 어긋나면 `40000`). **중복 지급 방어 로직 필수**, 토스앱 5.232.0+, 사전 테스트 프로모션 코드 1회 호출 필요, userKey당 분당 10회 제한.

## 4. 포맷 — "오늘의 3종 예측"

매일 **O/X 3문제**를 한 세트로 낸다. (단일 종목이면 "5초 만에 끝나는 게임"이 됨 → 3종이면 체류 3배 + 광고 자리 ↑ + 매일 콘텐츠 풍부)

```
오늘의 전국민 예측
 ① 달러 오른다?      O / X
 ② 코스피 오른다?    O / X
 ③ (오늘의 변동 슬롯) O / X
 → 오늘 점수 / 적중 현황
```

**슬롯 구성**
- **고정 2종**: 환율(USD/KRW), 코스피 — 전국민이 매일 신경 쓰는 주제라 고정
- **로테이션 1종**: 날씨 / 금값 / 유가 / 스포츠(국대전 등)를 날짜별로 교체 → 다양성으로 질림 방지

### 종목 후보 & 무료 API

| 종목 | 예측 | 정답 판정 | 무료 API | 추천 |
|---|---|---|---|---|
| 💵 **환율** | 내일 USD/KRW 오를까? | 다음 날 시세 > 오늘 | **한국수출입은행 환율 API** (공공데이터) | ★★★★★ |
| 📈 **코스피** | 내일 코스피 오를까? | 내일 종가 > 오늘 종가 | 한국거래소(KRX) / 야후파이낸스 | ★★★★☆ |
| 🌧️ **날씨** | (아래 *전국 단위*로) | 기상 데이터 | **기상청 OpenAPI** | ★★★★★ |
| 🥇 금값/유가 | 내일 오를까? | 시세 비교 | 공공데이터/오픈 API | ★★★★☆ |
| ⚽ 스포츠 | 손흥민 팀 이길까? 등 | 경기 결과 | API-Football 등(일부 무료) | ★★★☆☆ |

> 정답 판정은 사실 API보다 단순함 — API는 **숫자만 가져오고**, 게임은 `오늘값 > 어제값 ? "오른다" : "내린다"` 비교만 하면 끝. (스포츠만 경기취소·연장·승부차기 예외처리 있어 MVP엔 부담)

### ⚠️ 종목 선정 2원칙 (중요)
1. **전국민이 같은 문제를 푼다** — 랭킹이 성립하려면 필수. 유저별 개인화 금지.
2. **전국민이 다들 궁금해할 것** — 로컬 관심사는 탈락.
   - 그래서 날씨도 ❌"서울 비 올까"(로컬) 가 아니라 ✅**전국 단위**로: "내일 전국 첫눈 올까", "장마 시작될까", "내일 폭염주의보 뜰까", "내일 수도권 미세먼지 '나쁨'일까" → 지역 무관·위치 권한 불필요.

## 5. 점수 / 랭킹 설계

- **적중 +10점, 빗나감 0점** (감점 없음 = 부담 없이 매일 참여)
- **올킬 보너스 +20점**: 그날 정답이 나온 전 종목을 답하고 전부 적중하면 가산점 → "오늘 올킬 가자" 목표 + 3종 다 참여 유도
- **연속 적중 스트릭**: 3연속·5연속·7연속 보너스, 끊기면 0으로 리셋 → "스트릭 깨기 싫어서" 매일 옴
- **랭킹**: **토스 게임센터** 누적 점수 보드(친구·공유 내장). 주간/종목별 보드는 게임센터 다중보드 지원 여부에 따라(콘솔 확인) — 우선 전체 누적 랭킹으로 출시
- **통계**: 내 적중률 %, 전국민 평균 대비 → "나 환율은 잘 맞히네" 같은 정체성
- **뱃지/업적**: 첫 적중, 10연속, 적중률 70% 돌파 등 수집 요소

## 6. 광고 배치 (수익 포인트)

수익원은 **인앱 광고 2.0 ver2**(토스 애즈+AdMob 통합). 전면형/리워드형은 `loadFullScreenAd`/`showFullScreenAd` 같은 API, `adGroupId`로 타입 결정. 배너는 별도.

| 위치 | 형태 | 명분(시청 완료 시 지급) |
|---|---|---|
| **어제 채점 결과 확인 직후** | 전면 | 자연스러운 전환 지점 (진입 직후 아님) |
| **스트릭 보호** | 리워드 | "광고 보면 오늘 틀려도 연속기록 유지" (이탈방지·opt-in 높음) |
| **점수 부스트** | 리워드 | "광고 보면 오늘 획득 점수 2배" |
| **예측 전 힌트** | 리워드 | "광고 보면 어제 추세/참고지표 보기" |
| **홈 하단 배너** | 배너 | 상시 (단 O/X 버튼과 충분히 이격) |

### 🚨 광고 정책 준수 (위반 시 정산 보류·환수·영구정지)
- **리워드는 `userEarnedReward`(시청 완료)에서만 지급.** `dismissed`로 지급 금지. 문구는 "광고 **보면**"(❌"클릭하면").
- **앱 진입 직후 전면광고 금지.** 결제·인증 흐름 방해 금지. 뒤로가기 차단/막다른 구조 금지.
- 배너는 **O/X 버튼 등 상호작용 요소와 인접 배치 금지**(오클릭 유도). 한 화면에 **같은 포맷 2개 이상 금지**. "ad" 표기·표준 컴포넌트 유지, 색상·문구 임의수정 금지.
- 자동 refresh·인위적 노출/클릭 금지.
- 개발 땐 **테스트 광고 ID 필수** (`ait-ad-test-interstitial-id` / `ait-ad-test-rewarded-id` / `ait-ad-test-banner-id`). 실제 ID로 테스트 = 위반.

> 점수 부스트/스트릭 보호는 가상 점수라 사행성·환금성과 무관. "클릭"이 아니라 "시청완료" 기준이라 정책 안전.

## 7. 화면 구성 (MVP)

하단 네비게이션 3탭 구조:
1. **🎯 오늘** (기본) — 상단 스트릭/총점 → 어제 결과 압축 카드(+2배 리워드) → **오늘의 현재 문제를 크게**(진행 칩 ✓로 표시, 종목 전환) → 예측 확정/다시 수정. *오늘 예측을 압도적으로 우선 노출.*
2. **📊 기록** — 총점·최고 스트릭·전체 적중률·종목별 적중률(바)
3. **🏆 랭킹** — `openGameCenterLeaderboard()` 로 토스 게임센터 웹뷰(친구/공유). ※ 앱 내 커스텀 아님

## 8. 데이터 저장 / 아키텍처 (결정됨 — 게임센터 기반, 서버 0개)

→ 상세 설계는 **`아키텍처.md`** 참고. 핵심만:

| 데이터 | 어디에 | 비용 |
|---|---|---|
| 랭킹(누적점수·친구·순위) | **토스 게임센터 리더보드** | 무료 (토스가 관리) |
| 오늘의 3문제 + 정답 | **GitHub Pages `today.json`** (GitHub Actions가 매일 갱신) | 무료 |
| 내 예측·누적점수·스트릭·종목별 적중률 | **토스 `Storage`** (로컬, 문자열만) | 무료 |

- **Firebase 안 씀.** 랭킹은 토스 게임센터가 통째로 제공(친구·공유 내장).
- 공개 API(환율/기상청 등)는 CORS·키노출 때문에 **GitHub Actions(서버사이드)** 가 호출 → 결과만 정적 JSON으로 게시 → 클라가 읽음.
- **대가**: 게임센터 = **게임 카테고리 전용 → GRAC 등급분류(10~15일) 필요**. 랭킹 UI는 토스 통제(별도 웹뷰). 전국민%·주간/종목별 보드는 제약(나중에 옵션).
- 게임센터 SDK: `submitGameCenterLeaderBoardScore`(점수 제출) / `openGameCenterLeaderboard`(랭킹 열기), 최소 토스앱 5.221.0+, 미니앱 승인 후 호출.

## 9. 기술 메모 (앱인토스)

- 순수 HTML/JS도 가능: Vite + `@apps-in-toss/web-framework`
- 빌드 `npx ait build` → `.ait` 콘솔 수동 업로드
- 광고·정산은 **사업자 등록 필요**, 리워드 광고 SDK 적용
- 게임 카테고리 출시 시 게임물 등급분류(GRAC) 필요 가능성 → "콘텐츠" 카테고리 검토
- 상세 함정은 `appsintoss-deploy-gotchas` 메모 참고

### 마감/정답 타이밍 (영업일 기준, 내일 결과 예측)
- **예측 대상 = 다음 영업일 결과** ("내일 환율 오를까?"). → "내일 확인하러 다시 옴" 재방문 루프.
- **예측 마감**: 매일(영업일) 고정 시각(예: 장 시작 전 오전 9시)까지 그날의 3문제 예측.
- **정답 확정**: 다음 영업일 장 마감 후(코스피 15:30 / 환율 마감시세) → GitHub Actions가 채점.
- **주말·공휴일**: 환율·코스피는 장이 안 열려 정답 불가 → 다음 영업일로 이월, 그 사이 로테이션 슬롯은 **24/7 종목(날씨 등)** 으로 대체.
