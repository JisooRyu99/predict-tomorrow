# 아키텍처 (채택안 — 토스 게임센터 기반, 무료)

> 결정(2026-06-25): 랭킹은 **토스 게임센터 리더보드**를 쓴다. Firebase 안 씀.
> (대안 B = 직접 Firebase 구현은 `firebase구조.md` 참고, 미채택.)

---

## 0. 기술 스택 (결정됨)

- **WebView 경로** = `@apps-in-toss/web-framework` + Vite (밸런스 게임과 동일, 코드/노하우 재활용).
- 따라서 광고는 **WebView API** 사용: 전면/리워드 `loadFullScreenAd`·`showFullScreenAd`, 배너 `TossAds.attachBanner`.
- ⚠️ React Native 경로(`@apps-in-toss/framework`)의 `InlineAd`는 **안 씀** (RN 전용 컴포넌트). 문서에서 InlineAd가 보이면 우리 경로 아님.

## 1. 전체 그림

```
[토스 미니앱 클라이언트]
   ├─ 오늘의 문제 = GitHub Pages의 today.json 읽기
   ├─ 유저가 3종 O/X 예측 → 토스 Storage(로컬)에 저장
   ├─ (다음 날) today.json의 정답으로 채점 → 점수 계산
   ├─ 누적 점수 → submitGameCenterLeaderBoardScore() 로 제출
   └─ openGameCenterLeaderboard() 로 랭킹 화면(친구/공유) 열기

[GitHub Actions 크론 (무료)]  ← 문제 출제 + 정답 확정
   매일 1회:
   1. 공개 API에서 환율/코스피/날씨 등 수치 조회
   2. 어제값과 비교해 정답 계산 (오늘값>어제값 ? "오른다":"내린다")
   3. 오늘의 3문제 + 어제 문제의 정답을 today.json 으로 생성
   4. GitHub Pages 에 커밋/배포 (정적 JSON, CORS 허용)

[토스 게임센터]  ← 랭킹 저장·UI·친구·공유 전부 토스가 관리
```

**서버·DB 0개.** GitHub Actions(무료) + GitHub Pages(무료) + 토스 SDK(무료)로 끝.

---

## 2. 왜 이 구조인가 (CORS 함정 회피)

- 한국수출입은행·기상청 등 **공개 API는 CORS 미허용**이 많아 클라이언트(웹뷰)에서 직접 못 부름.
- 그래서 **GitHub Actions(서버사이드 fetch, CORS 무관)** 가 대신 호출해 계산하고, 결과만 **GitHub Pages(CORS 허용)** 에 정적 JSON으로 올림 → 클라이언트는 이 JSON만 읽으면 됨. API 키도 클라이언트에 노출 안 됨.

---

## 3. 데이터 위치

| 데이터 | 어디에 | 비고 |
|---|---|---|
| 오늘의 3문제 + 정답 | **GitHub Pages `today.json`** | GitHub Actions가 매일 갱신 |
| 랭킹(누적 점수·친구·순위) | **토스 게임센터** | 토스가 관리, 직접 저장 안 함 |
| 내 예측·누적점수·스트릭·종목별 적중률 | **토스 `Storage`** (로컬) | 문자열만, `JSON.stringify` |

## 4. `today.json` 예시 (GitHub Pages)

```jsonc
{
  "date": "2026-06-25",
  "questions": [
    { "id": "exchange", "q": "내일 원/달러 환율, 오를까?", "options": ["오른다","내린다"] },
    { "id": "kospi",    "q": "내일 코스피, 오를까?",       "options": ["오른다","내린다"] },
    { "id": "weather",  "q": "내일 전국에 첫눈 올까?",     "options": ["온다","안온다"] }
  ],
  "answers": {                      // 어제(2026-06-24) 문제의 확정 정답
    "date": "2026-06-24",
    "exchange": "오른다",
    "kospi": "내린다",
    "weather": "안온다"
  }
}
```

> 한 파일에 **오늘 문제 + 어제 정답**을 같이 담으면, 클라가 한 번 읽어서 "어제 채점 → 오늘 예측"을 다 처리.

## 5. 게임센터 리더보드 SDK (핵심 사용법)

- **점수 제출**: `submitGameCenterLeaderBoardScore({ score: "1230" })` — score는 **문자열 숫자**.
- **랭킹 열기**: `openGameCenterLeaderboard()` — 토스 랭킹 웹뷰 표시(친구/공유).
- **버전 가드**: `isMinVersionSupported({ android: "5.221.0", ios: "5.221.0" })` 로 먼저 확인. 미만이면 `undefined`.
- **주의**:
  - 게임 진입 직후 말고 **플레이(예측 제출) 완료 후** 점수 제출.
  - **미니앱 정보 승인 전** 호출 시 `LeaderBoard not found`.
  - 랭킹 웹뷰 열면 **미니앱이 백그라운드** 전환 → 복귀 시 상태 관리 유의.
  - **서버 점수 검증 없음** → 점수 계산·유효성은 클라가 책임(가상 점수라 감수). 누적 점수는 토스 Storage에 보관하며 갱신.
  - 게임 **카테고리 전용**. (비게임에서 호출 시 동작 안 함)
  - 샌드박스 점수는 실제 리더보드 미반영.
- 예제: `apps-in-toss-examples` 레포의 `with-game`.

### ⚠️ 누적 점수 + 기기 변경 이슈
- 게임센터는 제출한 score를 **읽어오는 함수가 없음**(보안상 식별자도 응답 제외). 누적 점수의 원본은 **토스 Storage(로컬)** 에만 있음 → 기기 변경/재설치 시 로컬 누적 0으로 초기화.
- **회피**: 콘솔에서 리더보드를 **"최고 점수(best)" 모드**로 설정 → 누적이 단조 증가이므로, 기기 바뀌어 낮은 값 재제출해도 기존 최고기록이 유지되어 순위 안 깨짐. (출시 전 콘솔 설정 확인 필요)

## 6. 채점 로직 (클라이언트)

```js
// today.json 로드 후
const myBet = Storage.getItem("bet:2026-06-24");        // 어제 내 예측
const { answers } = todayJson;                           // 어제 정답
let gain = 0;
for (const id of ["exchange","kospi","weather"]) {
  if (JSON.parse(myBet).answers[id] === answers[id]) gain += 10;
}
const total = Number(Storage.getItem("totalScore") ?? 0) + gain;
Storage.setItem("totalScore", String(total));
submitGameCenterLeaderBoardScore({ score: String(total) });
```

스트릭/종목별 적중률도 같은 시점에 Storage에서 갱신.

## 6.5 광고(수익화) 구현 메모

- SDK: 인앱 광고 2.0 ver2. `loadFullScreenAd` → (`loaded` 이벤트) → `showFullScreenAd` → 다음 `load` 프리로드. 전면/리워드 동일 API, **`adGroupId`로 타입 결정**.
- **프리로드**: 버튼 클릭 시 로드 ❌ → 화면 마운트 시 미리 로드, 광고 닫힌(`dismissed`) 직후 다음 광고 로드.
- **리워드 지급은 `userEarnedReward` 이벤트에서만.** (`dismissed`만으로 지급 ❌). `impression` = 수익 발생 시점.
- **버전**: 5.247.0+ (토스애즈+AdMob), 5.227.0~ (AdMob 단독), 미만 미지원. `loadFullScreenAd.isSupported()`로 가드.
- **iOS**: App Tracking Transparency(ATT) 미허용 시 일부 광고 로드 실패.
- **언마운트 시 `unregister()`** 호출(메모리 누수 방지), `onError` 항상 제공.
- **테스트 ID**: 전면 `ait-ad-test-interstitial-id`, 리워드 `ait-ad-test-rewarded-id`, 배너 `ait-ad-test-banner-id`/`ait-ad-test-native-image-id`. 실제 ID로 테스트 금지.
- 정책 상세는 기획안 6장 참고.

### 배너 광고 (전면/리워드와 별개 API·버전)
- API: `TossAds.initialize({callbacks})` (앱 최상위에서 **1회만**) → `TossAds.attachBanner(adGroupId, el, {theme,tone,variant,callbacks})` → 반환 객체 `.destroy()`. 페이지 이동 시 `TossAds.destroyAll()`.
- **버전**: 배너는 **5.241.0+** (전면/리워드 ver2는 5.247.0 — 서로 다름!). 미만은 **빈 화면** → `TossAds.initialize.isSupported()` + 토스앱 버전 체크로 가드.
- **수익 시점**: `onAdViewable` (광고가 화면 **50% 이상·1초 이상** 보여야 집계). 스쳐 지나가면 미집계.
- **자동 refresh**: SDK가 렌더 후 10초 경과 시 / visibility false→true 시 자동 갱신 → **체류 긴 화면(홈)에 두면 노출 누적**. (우리가 임의 refresh 추가는 정책 위반)
- **컨테이너**: `width:100%`(화면 너비) 필수, 고정형 `height:96px` 권장. width를 px 고정 ❌. 부착 엘리먼트 내부는 비워둘 것.
- 언마운트 시 `destroy()` (메모리 누수 방지), `onInitializationFailed`/`onAdFailedToRender`/`onNoFill` 처리.
- 테스트 ID: 리스트형 `ait-ad-test-banner-id`, 피드형 `ait-ad-test-native-image-id`.
- 배치: **홈 하단**(체류 길어 refresh 수익 ↑), O/X 버튼과 이격. `useTossBanner` 커스텀 훅으로 재사용 권장.

## 7. 무료 유지 체크리스트

- [ ] Firebase·자체 서버 0개 (GitHub Actions + Pages + 토스 SDK만)
- [ ] 공개 API 호출은 전부 GitHub Actions(서버사이드)에서 → CORS·키노출 회피
- [ ] 리더보드 "최고 점수" 모드로 콘솔 설정 (기기변경 대비)
- [ ] 게임 카테고리 → **GRAC 등급분류 일정(10~15일)** 출시 계획에 반영
- [ ] 미니앱 정보 승인 후에 리더보드 호출 (LeaderBoard not found 방지)
- [ ] 버전 가드(5.221.0+) 적용

## 8. 남은 한계 / 나중 고려

- **전국민 % 집계**: 게임센터로는 불가. 정말 원하면 나중에 무료 카운터(Firebase 등) 하나만 얇게 추가.
- **주간/종목별 보드**: 게임센터 다중 보드 지원 여부 콘솔에서 확인 필요. 안 되면 전체 누적 랭킹만으로 출시.
